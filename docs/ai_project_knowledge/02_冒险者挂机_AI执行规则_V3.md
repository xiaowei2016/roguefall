## 9. 强制代码规则

### 9.1 命名规则
- 标识符必须英文，禁止中文标识符
- 节点名：snake_case（如 hp_bar, attack_timer）
- 脚本类名：PascalCase（如 BattleWidget, InputRegionManager）
- 信号名：snake_case（如 battle_widget_moved, panel_opened）

### 9.2 注释规则（硬性要求）
**必须加注释的位置：**
- 文件头：文件用途、依赖关系
- 关键逻辑入口：如 _ready()/核心循环/状态机转折点
- 边界条件：if分支覆盖的所有情况
- 防误改注释：硬编码数值/固定关系的来源与冻结原因（引用hash 0fd34a8）

**允许但非强制：** 中文注释说明逻辑，ASCII流程图/状态图。

### 9.3 禁止注释
- 复制粘贴的旧代码注释
- 与实现不符的过时注释
- 已废弃函数/变量的残留注释

## 10. 数据架构硬约束

### 10.1 两层极简架构

砍掉 TRES 中间层（不适合 CSV 批量导入场景），采用两层极简架构：

| 层级 | 组件 | 职责 |
|---|---|---|
| 第1层 | CSV 外置数据表 | 唯一数值来源，Excel 直接编辑导出 |
| 第2层 | 全局数据管理单例 | 游戏统一读取入口，启动一次性缓存全部数据 |

### 10.2 五大固定数据库

| 编号 | 数据库 | 核心字段 |
|---|---|---|
| 1 | 怪物数据库 | 等级 / 血量 / 防御 / 攻击 / 经验 / 掉落组ID |
| 2 | 掉落权重数据库 | 掉落组 / 物品ID / 权重 / 品质区间 / 掉落限制 |
| 3 | 装备基础数据库 | 部位 / 基础属性 / 等级 / 词条上限 / 成长率 |
| 4 | 词条数值数据库 | 词条ID / 类型 / 数值区间 / 等级限制 / 专属词条 |
| 5 | 图鉴与宠物数据库 | 图鉴6类数据 / 宠物属性加成比例 |

### 10.3 目录规范

```
res/
  data_csv/             ← 所有 CSV 配置表
  scripts/
    data_core/          ← 全局数据管理器（DataManager 单例）
```

TRES 资源目录不创建。

### 10.4 强制开发规则

1. 所有数据靠 ID 关联，禁止文字匹配、禁止硬编码数值
2. 修改怪物血量 / 掉率 / 装备属性只改 CSV 表格，禁止改代码
3. 游戏启动一次性缓存全部数据，全局读取统一走 DataManager 入口
4. 架构稳定可扩展，禁止推翻重来、禁止嵌套数据结构、禁止冗余表
5. 所有 UI / 战斗 / 挂机逻辑只读数据层，不生成不写死数据
