## 6. 口令长度限制与精简规则

GPT 给 Marvis / Codex 的任务口令必须控制长度。Marvis / Codex 执行端也应按本规则判断口令是否合规。

### 长度硬限制

- 普通任务：2500～3500 字。
- 复杂任务：最多 5000 字。
- 禁止超长口令。

### 内容精简规则

- 禁止把完整项目文档塞进口令。
- 禁止把大段历史对话塞进口令。
- 只列必读文档路径，不复制文档全文。
- 只写本次目标、本次禁止、执行步骤、验收标准、回执要求。
- 复杂背景只写必要摘要。
- 能用文档路径引用的内容，不要在口令中重复展开。

### 任务拆分规则

- 每个任务只做一个明确目标。
- 如果任务超过 5000 字，必须拆成多个小任务。

### 精简不可删除项

口令精简不能删掉以下硬规则：

- 项目路径
- 必读文档
- 本次目标
- 本次禁止
- 验证（headless / F5）
- commit
- 回执

### Godot 特殊保留

- 涉及 Godot 功能开发时，仍必须保留 Godot 官方文档复核要求。
- 涉及可见内容时，仍必须保留 headless + F5 验收要求。

## 7. 代码注释规则

### 7.1 基本要求

- **允许并推荐中文注释**，方便零基础项目所有者理解代码。
- 文件名、`class_name`、函数名、变量名、信号名、节点名默认使用**英文**。
- **绝对禁止**把代码标识符（函数名/变量名/信号名等）改成中文。

### 7.2 必须加注释的位置

1. **文件头**：文件名、功能说明、所属场景、节点树关系、对应文档路径。
2. **关键逻辑**：非显而易见的状态机、条件分支、循环体。
3. **边界条件**：`<=` vs `<`、`>=` vs `>` 等易混淆判断，说明为何选这个边界。
4. **复杂逻辑**：超过 10 行的函数体，用简单的 ASCII 流程图在注释中说明走向。
5. **防误改说明**：对冻结的固定值、锚点比例、硬编码禁区，加注释标注 `【冻结】` 或 `【禁止硬编码】`。

### 7.3 ASCII 流程图示例

```gdscript
# 面板切换状态机：
#   [点击按钮] → 当前面板显示？
#                  ├─ 是 → 隐藏面板 → [结束]
#                  └─ 否 → 显示面板 → 刷新穿透区域 → [结束]
func toggle_panel():
    ...
```

### 7.4 注释风格

- 单行注释用 `# `（井号+空格），段注释用 `# -----` 分隔。
- 参数说明用 `# 【参数名】说明文字`。
- 触发时机用 `# 【触发时机】场景就绪 / 按钮按下 / 信号连接 ...`。
- 冻结标记用 `# 【冻结】此值禁止修改，见文档 03 §1.4`。
