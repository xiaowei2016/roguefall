## 2. 游戏定位（不可违背）

**2D横版桌面挂机RPG。** 透明桌面窗口，角色在战斗条中自动战斗/成长/拾取。玩家核心操作是查看装备、换装、洗练词条、查看图鉴、培养宠物、切换地图。

本项目是：桌面小窗口挂机、角色长期成长、刷装备与词条、收集图鉴与宠物、轻量单机。
本项目不是：手操动作、开放世界、多人联网、商城驱动、大型剧情RPG。

核心循环：自动寻怪 → 自动战斗 → 怪物死亡 → 掉落奖励 → 属性成长 → 解锁更强内容 → 持续挂机。

## 3. 平台与窗口总规则

- 第一阶段只做Windows桌面版
- 主窗口：1440×718 透明逻辑画布，BattleBar 720×180 常驻挂机条
- main.gd 统一管理：窗口属性、BattleBar拖拽、面板布局（翻转让位）、状态机、持久化
- DesktopPixelPassthrough.cs 负责 Win32 像素级鼠标穿透（按像素 alpha 动态切换 WS_EX_TRANSPARENT）
- CenterPanel 从 BattleBar.Button("背包") toggle 打开，LeftPanel/RightPanel 随 CenterPanel 联动
- 窗口核心数学冻结 hash：0fd34a8，常量定义见 03号文档第9节

旧架构已废弃：BattleStrip/layout_shell/动态UI创建/页面脚本控制穿透/WindowShell/DockLayoutController/InputRegionManager。禁止回流。

## 4. Godot 可视化强制规则（最高优先级）

凡是玩家能看到的东西，必须用Godot可视化资源：.tscn / Sprite2D / AnimatedSprite2D / Control / TextureRect 等制作。

脚本只允许：数据读取、数值刷新、显示隐藏、按钮事件、动画播放控制、状态切换、拖动逻辑、布局计算、穿透区域更新、战斗计算、掉落判定、存档读写。

脚本禁止：用代码new出完整UI面板/装备格子/角色外观/怪物外观/地图背景/动画结构。

## 5. 美术方向

彩虹岛/LaTale Q版氛围：明亮柔和、2.5-3头身、横版童话冒险感、色彩鲜艳不过饱和、线条清晰、平涂加轻微柔和阴影。

AI生图必须用纯品红 #FF00FF 背景，禁止任何伪透明/棋盘格/格子背景，品红背景由美术后期一键去底。透明素材（非AI生成）必须真透明PNG（Alpha通道）。禁止海报/概念图作为游戏资产。

禁止风格：像素风、暗黑写实、厚涂重油画、儿童贴纸感、写实比例角色。

## 6. 游戏核心设计规则

### 6.1 生态地图铁则

地图是生态世界，不是刷波次副本。禁止：波次/阶段/玩家升级后怪物同步升级/同屏数量限制决定存在。正确：固定WorldEntry，等级门控激活（monster_level <= player_level + 3），怪物等级决定掉落。

每张常规地图：5种普通怪 + 1种精英怪 + 1个Boss = 7种生物。详情见 08号文档。

### 6.2 装备系统铁则

装备生成只能从固定词条库抽取（16个mod_id），禁止临时硬编码随机属性。flat_attack与percent_attack是不同的mod_id。品质只决定词条数量（白0/绿1/蓝2/金3/红4），不放大基础属性。同一mod_group不重复出现。不设百分比词条数量上限。详情见 07号文档。

### 6.3 战斗铁则

自动战斗，玩家不需手动攻击。non_engage是AI即时判断（不写入存档），sleeping是持久休眠状态。普通怪接战上限3只，精英接战上限1只。

### 6.4 UI铁则

三层节点封顶（Control → 容器 → 原子组件）。独立.tscn预制文件。全局Theme主题。禁止多层嵌套。自由拖拽位置。详情见 03号文档。
