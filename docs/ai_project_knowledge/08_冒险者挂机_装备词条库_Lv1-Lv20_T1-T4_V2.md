# 冒险者挂机 装备词条库 V2：Lv1-Lv20 / T1-T4（完整保留版）

> 更新时间：2026-06-21  
> 用途：作为 冒险者挂机 装备随机词条系统的当前项目来源文件。本文完整保留 Lv1-Lv20 / T1-T4 词条、权重、部位、品质、洗练预留等数值。  
> 替代：旧版 `roguefall_equipment_affix_database_v1_1.md`。

## 1. 核心规则

冒险者挂机 装备系统采用固定词条库。

装备掉落时，不允许临时随机生成"攻击可能是固定值，也可能是百分比"这种模糊词条。

正确规则：

| mod_id | 含义 |
|---|---|
| `flat_attack` | 固定攻击词条：攻击 +X |
| `percent_attack` | 攻击百分比词条：攻击 +X% |

二者是不同的 mod_id。  
装备可以同时拥有固定攻击和攻击百分比，但必须分别来自词条库。

每个词条必须由数据库固定：

```text
mod_id
display_name
stat_key
value_type
tier_values
allowed_slots
weight
mod_group
tags
```

## 2. 装备生成流程

1. 确定装备部位。
2. 确定装备等级 iLv。
3. 根据 iLv 计算 T 阶。
4. 根据装备品质确定随机词条数量。
5. 从该部位可用词条库中筛选符合 T 阶的 mod_id。
6. 按权重抽取具体 mod_id。
7. 根据该 mod_id 的 T 阶范围 roll 数值。
8. 写入装备实例。

禁止：

- 代码里临时写 `rand_attack_or_percent`
- 一个词条同时可能是 flat 也可能是 percent
- 装备生成时临时拼接未知 stat_key
- 绕过词条库直接写随机属性

## 10. 抽取限制

### 10.1 同组限制

同一件装备中，同一个 `mod_group` 最多出现一次。

允许：

```text
flat_attack
percent_attack
```

不允许：

```text
flat_attack
flat_attack
```

### 10.2 百分比词条限制

V1 版本不设百分比词条数量上限。装备品质仅控制随机词条总数（见第 3 节），不再额外限制其中百分比词条的数量。装备可出现的词条完全由其部位词条池决定。

> 设计说明：取消上限的原因是，V1 词条池中真正的约束来自 §10.1 的同组限制——同一 `mod_group` 不会重复出现。固定值词条（如 `flat_attack`）和百分比词条（如 `percent_attack`）分属不同的 `mod_group`，两者可以同时出现在同一件装备上。品质词条总数已提供了一层自然的数量约束，额外再加百分比上限会导致高品装备词条池可选范围过窄，反而降低多样性。

### 10.3 部位限制

装备只能从自己部位允许的词条池中抽词条。

例如：

- 武器不能出 `flat_hp`
- 衣服不能出 `crit_damage`
- 徽章不能出 `flat_attack`
- 鞋子可以出 `move_speed`
- 耳环可以出 `attack_speed`

## 12. 属性计算顺序

最终属性计算顺序：

```text
最终属性 =
角色基础属性
+ 装备基础固定属性
+ 随机固定值词条
再乘随机百分比词条
```

例子：

```text
角色攻击 = 56
武器基础攻击 = +32
戒指基础攻击 = +14
flat_attack 词条 = +10
percent_attack 词条 = +5%

最终攻击 = (56 + 32 + 14 + 10) × 1.05
```

## 14. 未来洗练系统预留：不进入 V1 实装

以下是已经确认过的未来设计，但不进入当前 V1 掉落装备实现。

### 14.1 未来目标

未来洗练系统可以扩展到：

```text

```

但当前 V1 掉落装备仍按品质生成 0~4 条词条。

### 14.2 洗练石规则预留

未来洗练：

```text
每次洗练随机重置未锁定的 1~6 条词缀
```

锁定消耗：

| 锁定数量 | 单次洗练消耗洗练石 |
|---:|---:|
| 锁 1 条 | 1 |
| 锁 2 条 | 3 |
| 锁 3 条 | 7 |
| 锁 4 条 | 15 |
| 锁 5 条 | 31 |

说明：

- 锁 5 条洗 1 条，单次消耗 31 个洗练石。
- 洗练系统必须等固定词条库与装备生成稳定后再接入。
- 不允许洗练临时生成词条库外 mod_id。

## 15. 验收标准

1. 装备生成不硬编码随机属性。
2. `flat_attack` 与 `percent_attack` 是不同 mod_id。
3. 品质只决定词条数量，不放大基础属性。
4. 白装有基础属性但无随机词条。
5. 同一 mod_group 不重复出现。
6. 部位只能抽对应允许词条。
7. Boss 掉落 iLv21-iLv23 时临时沿用 T4。
8. 未来洗练系统不会破坏固定词条库。

## 16. V1 设计原则补充说明

装备词条 V1 优先保证"玩家能感受到属性变化"。T1 词条不能低到完全无感，收益类词条也不能低到几乎抽不到。后续可以通过实测再回调数值。

基于此原则，v1.2 做了以下调整：

- **T1 低感知词条增强**：`move_speed` / `attack_speed` / `crit_chance` 的 T1 下限从 +1% 提升到 +1%~2% 或 +2%~3%，确保新手阶段换装后有可感知的提升。
- **收益类词条权重提升**：`exp_gain` / `gold_gain` / `item_drop` / `boss_damage` / `elite_damage` 权重统一提高到 65，避免因权重过低导致玩家十几件装备都见不到一条收益词条。
