# 冒险者挂机 装备词条库 V3：Lv1-Lv20 / T1-T4

> 项目名：冒险者挂机 | 引擎：Godot 4.7 | 路径：D:\Projects\roguefall
> 用途：装备随机词条系统的当前项目来源文件
> 核心原则：装备生成只能从固定词条库抽取，不允许临时硬编码随机属性

---

## 1. 核心规则

冒险者挂机装备系统采用固定词条库。装备掉落时不允许临时随机生成模糊词条。

| mod_id | 含义 |
|---|---|
| `flat_attack` | 固定攻击词条：攻击 +X |
| `percent_attack` | 攻击百分比词条：攻击 +X% |

二者是不同的mod_id。装备可同时拥有两者，但必须分别来自词条库。

每个词条必须由数据库固定：mod_id / display_name / stat_key / value_type / tier_values / allowed_slots / weight / mod_group / tags。

---

## 2. 装备生成流程

1. 确定装备部位 → 2. 确定装备等级 iLv → 3. 根据 iLv 计算 T 阶 → 4. 根据品质确定随机词条数量 → 5. 从该部位可用词条库筛选符合 T 阶的 mod_id → 6. 按权重抽取具体 mod_id → 7. 根据 mod_id 的 T 阶范围 roll 数值 → 8. 写入装备实例。

禁止：代码临时写 rand_attack_or_percent、一个词条同时可能是 flat 也可能是 percent、绕过词条库直接写随机属性。

## 10. 抽取限制

### 10.1 同组限制
同一件装备中，同一个 mod_group 最多出现一次。允许 flat_attack + percent_attack，不允许 flat_attack + flat_attack。

### 10.2 百分比词条限制
V1版本不设百分比词条数量上限。品质仅控制随机词条总数，不额外限制其中百分比词条数量。约束来自同组限制——同一 mod_group 不会重复出现，固定值词条和百分比词条分属不同 mod_group，可同时出现。

### 10.3 部位限制
装备只能从自己部位允许的词条池中抽词条。例如：武器不能出 flat_hp，衣服不能出 crit_damage，徽章不能出 flat_attack。

## 11. 第一版暂不开放的词条

吸血、闪避、减伤、抗暴、格挡、元素伤害、异常状态、技能等级、召唤物加成、离线收益倍率、自动拾取范围、药水效果提升。

---

## 12. 属性计算顺序

```
最终属性 = (角色基础属性 + 装备基础固定属性 + 随机固定值词条) × 随机百分比词条
```

例：角色攻击56 + 武器基础32 + 戒指基础14 + flat_attack词条10 = 112，再 × 1.05(percent_attack 5%) = 117.6。

## 13. 代码实现要求

必须建立 data/equipment_affix_database.gd 或等价数据表。装备生成时只能读取这个数据库，不允许临时硬编码随机属性。

## 14. 未来洗练系统预留（不进入 V1 实装）

未来洗练：装备最多6条词条，每次随机重置未锁定词缀。

| 锁定数量 | 单次洗练消耗洗练石 |
|---|---|
| 锁 1 条 | 1 |
| 锁 2 条 | 3 |
| 锁 3 条 | 7 |
| 锁 4 条 | 15 |
| 锁 5 条 | 31 |

洗练系统必须等固定词条库与装备生成稳定后再接入，不允许洗练临时生成词条库外 mod_id。

---

## 15. 验收标准

1. 装备生成不硬编码随机属性
2. flat_attack 与 percent_attack 是不同 mod_id
3. 品质只决定词条数量，不放大基础属性
4. 白装有基础属性但无随机词条
5. 同一 mod_group 不重复出现
6. 部位只能抽对应允许词条
7. Boss掉落 iLv21-iLv23 时临时沿用 T4
8. 未来洗练系统不会破坏固定词条库
