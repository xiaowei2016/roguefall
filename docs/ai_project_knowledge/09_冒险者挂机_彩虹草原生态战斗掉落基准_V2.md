## 0. 当前核心结论

冒险者挂机 每张常规地图采用：

```text
5 种普通怪 + 1 种精英怪 + 1 个 Boss = 7 种生物生态结构
```

当前彩虹草原采用：

| 层级 | 数量 | 当前彩虹草原物种 | 说明 |
|---|---:|---|---|
| 普通生态层 | 90 只 WorldEntry | slime / mushroom / clover_hopper / dandelion_sprite / moss_shell_snail | 5 种普通怪共同覆盖 Lv1-Lv19 |
| 精英生态层 | 3 只 WorldEntry | elite_wolf | 1 种精英怪，3 个真实精英 entry |
| Boss 层 | 1 只 WorldEntry | boss_forest_boar / 彩虹草原 Boss | 独立存在，不占普通池，不占精英池 |

总生态目标：

```text
90 normal + 3 elite + 1 boss = 94 WorldEntry
```

当前目标不是“屏幕刷怪”，而是：

```text
怪物存在归 WorldEntry。
怪物显示归等级门控 + 视野/实例化规则。
怪物接战归 AI 感知范围与 attacker slot。
```

## 1. 不可违背的设计原则

### 1.1 禁止做成波次 / 阶段

禁止出现：

- 第 1 波 / 第 2 波
- 阶段 1 / 阶段 2
- 玩家升级后，所有怪物同步升级
- 到某等级后强制换刷怪阶段
- 为了维持同屏数量，在玩家眼前凭空刷怪或删怪

正确感觉：

- 同一张地图里固定存在不同等级、不同种类的 WorldEntry。
- 玩家等级越高，当前可激活的生态越完整。
- 低等级时怪少、安全、目标清晰。
- 高等级时怪物逐渐变多，挂机效率和刷怪爽感提高。
- 怪物等级决定掉落装备等级。
- Boss 独立存在，不参与普通生态池。

一句话：

```text
冒险者挂机 的地图是生态世界，不是刷波次副本。
```

### 1.2 三层语义必须分离

| 概念 | 含义 | 允许限制吗 |
|---|---|---|
| 存在 | WorldEntry 数据是否存在 | 不允许用同屏数量限制，94 个 entry 固定存在 |
| 激活 | entry 当前是否允许实例化 / 显示 / 参与目标扫描 | 由 `monster_level <= player_level + 3` 决定 |
| 接战 | 是否主动追击 / 攻击玩家 | 允许用 AI、等级差、attacker slot 严格限制 |

禁止用以下旧概念决定怪物是否存在：

- `MAX_ENEMIES`
- `MAX_ACTIVE_INSTANCES`
- `TARGET_VISIBLE_ENEMIES`
- `MAX_ELITE_ON_SCREEN`
- `_density_spawn`
- `spawn_field_enemy`
- `_spawn_entry_force`

## 2. 地图通用生态公式

### 2.1 一张图 7 种生物

每张常规地图默认由：

| 类型 | 数量 | 说明 |
|---|---:|---|
| 普通怪物种 | 5 种 | 覆盖本地图主等级段 |
| 精英怪物种 | 1 种 | 装备检查点 / 高价值压力点 |
| Boss | 1 个 | 本地图通关检查点 |

合计：

```text
5 normal species + 1 elite species + 1 boss = 7 biological types
```

### 2.2 普通怪 5 段等级覆盖公式

每张地图的 5 种普通怪按“等级带重叠覆盖”设计。  
这不是写死某个怪物，而是通用公式。

以第一张图 Lv1-Lv19 普通生态为例：

| 普通槽位 | 等级覆盖 | 生态定位 |
|---|---:|---|
| Normal Slot 1 | Lv1-Lv6 | 新手 / 前期主怪 |
| Normal Slot 2 | Lv4-Lv10 | 前中期主怪 |
| Normal Slot 3 | Lv7-Lv13 | 中期主怪 |
| Normal Slot 4 | Lv10-Lv16 | 中后期压力怪 |
| Normal Slot 5 | Lv13-Lv19 | Boss 前普通压力怪 |

重点：

- 不是每个等级平均铺满。
- 不是指定某个怪永远是 Slot 1。
- 每张地图可以把不同物种填进 5 个普通槽位。
- 相邻槽位有 3 级左右重叠，让生态过渡自然。
- 怪物等级在 entry 创建时固定，不随玩家等级变化。
- 是否激活由玩家等级公式决定，不由 slot 解锁表决定。

### 2.3 彩虹草原当前普通槽位映射

| 普通槽位 | 彩虹草原当前物种 | 等级覆盖 | 说明 |
|---|---|---:|---|
| Normal Slot 1 | slime | Lv1-Lv6 | 前期弱怪 / 新手生态 |
| Normal Slot 2 | mushroom | Lv4-Lv10 | 前中期主挂机怪 |
| Normal Slot 3 | clover_hopper | Lv7-Lv13 | 中期活跃跳跃怪 |
| Normal Slot 4 | dandelion_sprite | Lv10-Lv16 | 中后期飘浮 / 远程感怪 |
| Normal Slot 5 | moss_shell_snail | Lv13-Lv19 | 后期慢速高防普通压力怪 |

说明：

- `moss_shell_snail` 当前素材可先作为测试版接入，后续额度恢复后可重做动画。
- 5 个普通槽位是公式；当前彩虹草原只是这个公式的第一张图实例。

## 3. 等级门控生态激活公式

### 3.1 唯一激活公式

当前采用全地图通用公式：

```text
可激活等级上限 = 玩家等级 + 3
```

即：

```gdscript
entry.monster_level <= player_level + 3
```

这个公式适用于：

- 普通怪
- 精英怪
- Boss

不使用 slot 解锁表。  
不使用“Slot 1/2/3/4/5 在固定玩家等级解锁”的写死规则。

### 3.2 正确理解

94 个 WorldEntry 在地图初始化时已经固定生成：

```text
entry_id
monster_id
monster_level
world_x
spawn_x
role
```

玩家等级只决定：

```text
这个 entry 当前能不能实例化 / 显示 / 被目标扫描到
```

玩家等级不改变：

```text
monster_id
monster_level
entry_id
world_x
spawn_x
drop_level
```

### 3.3 示例

| 玩家等级 | 可激活等级上限 | 当前效果 |
|---:|---:|---|
| Lv1 | Lv4 | 激活 Lv1-Lv4 entry |
| Lv3 | Lv6 | 激活 Lv1-Lv6 entry |
| Lv6 | Lv9 | 激活 Lv1-Lv9 entry |
| Lv10 | Lv13 | 激活 Lv1-Lv13 entry |
| Lv13 | Lv16 | 激活 Lv1-Lv16 entry |
| Lv16 | Lv19 | 普通怪 Lv1-Lv19 基本全部开放 |
| Lv17 | Lv20 | Boss Lv20 可开放 |

这样：

- 低等级开局怪更少、更清晰。
- 升级后地图逐步丰富。
- 后期 93 怪 + Boss 全部开放，刷怪更爽。
- Boss 不会在 Lv1 被自动锁定。
- 高等级怪不会太早出现后触发避战/离场问题。

### 3.4 禁止错误实现

禁止：

```text
slot_unlock_level
Slot1 Lv1 解锁
Slot2 Lv3 解锁
Slot3 Lv6 解锁
Slot4 Lv9 解锁
Slot5 Lv12 解锁
```

原因：

```text
用户要的是通用公式：monster_level <= player_level + 3
不是给 slot 写玩家等级解锁表。
```

## 4. 当前彩虹草原生态结构

### 4.1 当前实现目标：90 + 3 + 1

| 类别 | 数量 | 物种 | 等级范围 |
|---|---:|---|---:|
| 普通怪 Slot 1 | 18 | slime | Lv1-Lv6 |
| 普通怪 Slot 2 | 18 | mushroom | Lv4-Lv10 |
| 普通怪 Slot 3 | 18 | clover_hopper | Lv7-Lv13 |
| 普通怪 Slot 4 | 18 | dandelion_sprite | Lv10-Lv16 |
| 普通怪 Slot 5 | 18 | moss_shell_snail | Lv13-Lv19 |
| 精英怪 | 3 | elite_wolf | 建议 Lv11 / Lv14 / Lv17 |
| Boss | 1 | boss_forest_boar / 彩虹草原 Boss | Lv20 |

普通怪合计：

```text
18 × 5 = 90
```

总目标：

```text
90 + 3 + 1 = 94
```

### 4.2 等级生成方式

推荐实现：

```text
每个普通槽位定义：
species_id
level_min
level_max
entry_count
stat_profile
ai_profile
drop_profile
```

生成 WorldEntry 时：

1. 根据地图定义读取 5 个 normal_slot。
2. 每个 slot 生成 `entry_count` 个 WorldEntry。
3. 每个 entry 的 `monster_level` 在该 slot 的 `level_min ~ level_max` 内按稳定规则生成。
4. `monster_level` 创建后固定。
5. 不随玩家等级变化。
6. 不在运行时因为刷新而改变身份。
7. 复活是同一个 entry 恢复，不是新怪替换。

等级分配可以使用：

```text
level = level_min + (index % (level_max - level_min + 1))
```

或带权重的固定分布表。  
但禁止每次复活重新随机成另一个等级。

### 4.3 未来地图复用方式

第二张地图、第三张地图可以沿用同样公式，只整体平移等级段。

示例：

| 地图 | Slot 1 | Slot 2 | Slot 3 | Slot 4 | Slot 5 | Elite | Boss |
|---|---:|---:|---:|---:|---:|---:|---:|
| 彩虹草原 | 1-6 | 4-10 | 7-13 | 10-16 | 13-19 | 约 11/14/17 | 20 |
| 第二张图示例 | 20-26 | 23-29 | 26-32 | 29-35 | 32-38 | 约 30/33/36 | 39/40 |
| 第三张图示例 | 39-45 | 42-48 | 45-51 | 48-54 | 51-57 | 约 49/52/55 | 58/59 |

实际地图等级可按体验微调，但结构保持：

```text
5 个普通槽位递进覆盖 + 1 精英槽 + 1 Boss
```

激活公式也保持：

```text
entry.monster_level <= player_level + 3
```

## 5. WorldEntry 数据模型

每只普通怪、精英怪、Boss 都必须有独立 entry。

建议字段：

```text
entry_id
monster_id
monster_role
normal_slot
monster_level
is_elite
is_boss
max_hp
current_hp
attack
defense
gold_reward
state
activation_state
unlocked
world_x
spawn_x
respawn_timer
active_instance
```

### 5.1 entry_id 规则

当前建议：

```text
slime_00 ~ slime_17
mushroom_00 ~ mushroom_17
clover_hopper_00 ~ clover_hopper_17
dandelion_sprite_00 ~ dandelion_sprite_17
moss_shell_snail_00 ~ moss_shell_snail_17
elite_wolf_00 ~ elite_wolf_02
boss_forest_boar_00
```

规则：

- entry_id 一旦生成，不可重建成新身份。
- 禁止使用 `duplicate()` 复制 entry 作为新怪。
- 禁止死亡后生成一个“随机新精英”替代原精英。
- 禁止复活后变成另一个等级。
- 复活是同一个 entry 恢复，不是新怪替换。

### 5.2 状态规则

建议将“等级未开放”与“临时离场”分开。

| 状态 / 字段 | 含义 |
|---|---|
| `locked` / `unlocked=false` | 等级未开放，WorldEntry 存在但不实例化 |
| `active` | 已开放并实例化，可以巡逻 / 显示 / 接战 |
| `sleeping` | 已开放，但临时休眠 / 离场，可恢复 |
| `dead` | 已死亡，等待 respawn_timer |
| `respawning` | 可选状态；当前可不使用 |

重点：

- locked 不是 dead。
- locked 不是 sleeping。
- locked 不参与 respawn。
- locked 不参与 target scan。
- locked 不应该被删除。
- 玩家升级后重新评估 locked entry。

关于 `non_engage`：

- `non_engage` 不是怪物状态，不进入状态机，不写入存档。
- 它只是玩家 AI 在选择目标时的即时判断结果：当怪物等级过低（`level_diff <= -10`）或过高（`level_diff >= +7`）时，玩家 AI 跳过该目标。
- 下次扫描目标时重新判断。
- `sleeping` 才是生态系统里的持久休眠状态，可持久化。

## 6. 生态初始化与激活流程

### 6.1 初始化

```text
build_all_entries_shuffled(MAP_WIDTH)
→ 创建 90 normal + 3 elite + 1 boss = 94 WorldEntry
→ shuffle 全部 entry
→ 均匀分布 world_x / spawn_x 到 100 ~ 3500
→ 根据 player_level 调用 _update_ecology_activation(player_level)
→ 只实例化已开放 entry
```

### 6.2 分布规则

当前阶段建议：

```text
94 个 entry 均匀分布在 100 ~ 3500。
```

要求：

- 先创建全部 entry。
- 先 shuffle。
- 再分段均匀分配 world_x。
- 不按物种顺序铺点。
- 不让 slime 全在左侧、蜗牛全在右侧。
- Boss / 精英也应该有固定位置。
- locked entry 也有固定位置，只是暂不实例化。

### 6.3 激活更新

每次玩家等级变化后调用：

```gdscript
_update_ecology_activation(player_level)
```

逻辑：

```gdscript
func _is_entry_unlocked(entry: Dictionary, player_level: int) -> bool:
    return int(entry.get("monster_level", 1)) <= player_level + 3
```

升级后：

1. 遍历 normal entries。
2. 遍历 elite entries。
3. 遍历 boss entries。
4. 如果 locked entry 满足公式：
   - 设置 unlocked=true。
   - 允许实例化。
   - 不改变 entry_id。
   - 不改变 monster_id。
   - 不改变 monster_level。
   - 不改变 world_x / spawn_x。
5. 如果 entry 已经 unlocked，不建议重新锁回去。
6. 不要在玩家视野内无动画弹出；可延迟到视野外，或后续加出现动画。

## 7. 普通怪生命周期

### 7.1 巡逻 / 索敌 / 接战

普通怪 AI 必须满足：

| 阶段 | 规则 |
|---|---|
| 巡逻 | 围绕自己的 patrol_origin / spawn_x 活动 |
| 感知 | 只有玩家进入 `aggro_range` 才能主动追击 |
| 追击 | 进入 chase 前必须拿到普通 attacker slot |
| 攻击 | 进入 attack 前必须仍在 slot 内 |
| 放弃 | 超出 `leash_range` 必须释放 slot 并回巡逻 |
| 没拿到 slot | 不允许继续贴身挤玩家，应该巡逻 / 等待 / 停留 |

普通怪接战上限：

```text
MAX_ATTACKERS = 3
```

含义：

```text
最多 3 只普通怪主动追击 / 攻击玩家。
不是最多显示 3 只普通怪。
```

### 7.2 死亡与复活

普通怪死亡：

```text
entry.state = "dead"
entry.current_hp = 0
entry.active_instance = null
entry.respawn_timer = 3.0 ~ 8.0
```

普通怪复活：

```text
respawn_timer <= 0
entry 已 unlocked
→ current_hp = max_hp
→ 在 entry.world_x / spawn_x 恢复
```

locked entry 不参与 respawn。

### 7.3 视野内复活规则

设计基准：

```text
普通怪可以在自己的 territory / world_x 复活，
但不允许无动画地在玩家眼前凭空弹出。
```

二选一实现：

1. 如果复活点在玩家视野内，则延迟复活到玩家离开视野。
2. 或者播放明确的地面钻出 / 孢子重生 / 草丛出现动画。

## 8. 精英怪生命周期

### 8.1 精英不是“最多显示 1 只”

当前精英层规则：

```text
精英总 entry = 3
精英真实存在 = 3
精英显示数量 = 由等级门控与实例化决定
精英接战数量 = 最多 1
```

禁止恢复：

```text
MAX_ELITE_ON_SCREEN
MAX_ACTIVE_ELITE_NEAR_PLAYER
```

精英未开放时不显示。  
精英已开放且在视野内，就应该显示。  
不能因为“同屏最多 1 只精英”把另外两只隐藏、刷掉、延迟。

### 8.2 精英接战上限

```text
MAX_ELITE_ATTACKERS = 1
```

含义：

```text
最多 1 只精英主动追击 / 攻击玩家。
不是最多 1 只精英存在。
不是最多 1 只精英显示。
```

### 8.3 精英死亡与冷却

精英死亡：

```text
entry.state = "dead"
entry.current_hp = 0
entry.active_instance = null
entry.respawn_timer = 120.0
```

精英冷却：

```text
ELITE_RESPAWN_COOLDOWN = 120.0
```

规则：

## 10. 等级感知规则

`non_engage` 不是怪物状态，而是玩家 AI 选择目标时的即时判断——每次扫描目标时根据等级差重新计算，不写入存档也不进入怪物状态机。`sleeping` 才是生态系统里的持久休眠状态。

```text
等级差 = 怪物等级 - 玩家等级
```

| 等级差 | 行为 | 说明 |
|---:|---|---|
| `<= -10` | 怪物害怕逃跑 | 低级怪遇到高级玩家可逃跑 |
| `-9 ~ -6` | 玩家无视 | 不主动攻击，避免浪费挂机效率 |
| `-5 ~ +3` | 正常挂机目标 | 玩家默认自动攻击 |
| `+4 ~ +6` | 危险目标 | 默认绕开 |
| `+7 ~ +9` | 高危目标 | 默认绕开 |
| `>= +10` | 太危险 | 强制绕开 / 后退 |

默认挂机目标范围：

```text
玩家等级 -5 到 玩家等级 +3
```

注意：

- 等级门控已经让大部分高等级怪未开放。
- 已开放但仍偏高的怪，仍由 can_engage 控制。
- 高等级怪不应该因为低级玩家靠近而永久消失。
- 正确行为是玩家避开高等级怪，不是高等级怪离场 sleeping。
- `non_engage` 是即时 AI 判断，不影响怪物状态；`sleeping` 才是持久休眠状态。

## 14. 攻防与受击反馈

### 14.1 基础公式

```text
防御比 = 被攻击者防御 ÷ 攻击者攻击
```

双向适用：

- 玩家被怪打：玩家防御 ÷ 怪物攻击
- 怪物被玩家打：怪物防御 ÷ 玩家攻击

### 14.2 普通单位受击表

| 防御比 | 结果 | 表现 | 伤害倍率 |
|---:|---|---|---:|
| `< 0.30` | 重创 | 大后仰、僵直、当前攻击被打断 | 100% |
| `0.30 ~ 0.55` | 压制 | 小后仰、短暂停顿、攻击节奏被压一下 | 80% |
| `0.55 ~ 0.90` | 命中 | 只掉血，不明显后仰 | 60% |
| `0.90 ~ 1.40` | 扛住 | 原地承受，少量掉血 | 35% |
| `1.40 ~ 2.40` | 硬扛 | 很少掉血 | 15% |
| `> 2.40` | 化解 / 弹开 | 普通小怪攻击玩家时可 MISS | 0 或 1 点 |

### 14.3 Boss 攻击玩家受击表

| 防御比 | 结果 | 表现 | 伤害倍率 |
|---:|---|---|---:|
| `< 0.35` | Boss 重创 | 玩家大后仰，长僵直 | 100% |
| `0.35 ~ 0.65` | Boss 压制 | 玩家小后仰，短僵直 | 80% |
| `0.65 ~ 1.00` | Boss 命中 | 掉血，不明显后仰 | 60% |
| `1.00 ~ 1.60` | 扛住 Boss | 原地承受，少量掉血 | 35% |
| `1.60 ~ 2.80` | 硬扛 Boss | 很少掉血 | 15% |
| `> 2.80` | 化解 Boss | 极低伤害，但不显示 MISS | 1 点或 5% |

Boss 永远不显示 MISS。

## 17. 掉落规则

### 17.1 装备等级

普通怪 / 精英怪：

```text
掉落装备 iLv = 怪物等级 -2 到 怪物等级 +2
```

Boss：

```text
Boss 掉落装备 iLv = Boss 等级 +1 到 Boss 等级 +3
```

### 17.2 掉率

核心原则：

- 普通怪装备总掉率采用反向递减曲线：Slot1 最高，Slot5 最低。新手期更容易掉装备，后期掉装备变慢，让成长曲线更长。
- 这里的 Slot 是“本地图内部生态阶段”，不是全局等级含义；第二张地图的 Slot 1 也可以理解为该地图的新手段，因此可以沿用同一套 Slot 掉率节奏。
- Slot5 最低也保持 1.20%，不能低到没反馈。
- 普通怪所有 Slot（1-5）全部允许红色神装掉落，不只 Boss 才有红装。
- 装备品质在掉装后再判定。

**普通怪装备总掉率（反向递减曲线）**：

| 怪物 / 槽位 | 装备总掉率 | 定位 |
|---|---:|---|
| Slot 1 | 3.00% | 本地图新手段，高掉率，快速反馈 |
| Slot 2 | 2.50% | 本地图前中期 |
| Slot 3 | 2.00% | 本地图中期 |
| Slot 4 | 1.60% | 本地图中后期 |
| Slot 5 | 1.20% | 本地图后期压力怪，最低反馈线 |

**为什么普通怪总掉率前高后低**：

```text
这是刻意设计，不是写反。
Slot 1-2 掉装更频繁，用来保证每张地图刚进入时有装备反馈。
Slot 4-5 总掉率降低，用来拉长成长周期。
但 Slot 4-5 的金装/红装占比更高，用来保留后期“出货惊喜”。
```

**精英怪与 Boss 装备总掉率**：

| 怪物 | 装备总掉率 | 说明 |
|---|---:|---|
| 低级精英 | 12% | 对应本地图较早开放的精英 entry |
| 中级精英 | 15% | 对应本地图中段精英 entry |
| 高级精英 | 18% | 对应本地图后段精英 entry |
| Boss | 100% | 必掉 1 件装备 |

### 17.3 品质概率

装备品质在掉装后再判定，品质只影响随机词条数量，不放大基础属性。
