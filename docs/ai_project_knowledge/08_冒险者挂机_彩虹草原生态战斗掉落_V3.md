## 1. 不可违背的设计原则

### 1.1 禁止做成波次/阶段

禁止：第1波/第2波、阶段1/阶段2、玩家升级后怪物同步升级、强制换刷怪阶段、在玩家眼前凭空刷怪/删怪。

正确：同一张地图内固定存在不同等级/种类的WorldEntry。玩家等级越高，可激活的生态越完整。低等级怪少安全，高等级怪多刷得爽。怪物等级决定掉落装备等级。Boss独立存在。

> **冒险者挂机的地图是生态世界，不是刷波次副本。**

### 1.2 三层语义必须分离

| 概念 | 含义 | 限制方式 |
|---|---|---|
| 存在 | WorldEntry数据是否存在 | 不允许用同屏数量限制，94个entry固定存在 |
| 激活 | entry当前是否允许实例化/显示/扫描 | 由 `monster_level <= player_level + 3` 决定 |
| 接战 | 是否主动追击/攻击玩家 | 允许用AI、等级差、attacker slot严格限制 |

禁止用以下旧概念决定怪物是否存在：MAX_ENEMIES、MAX_ACTIVE_INSTANCES、TARGET_VISIBLE_ENEMIES、MAX_ELITE_ON_SCREEN、_density_spawn、spawn_field_enemy、_spawn_entry_force。

## 3. 等级门控生态激活公式

### 3.1 唯一激活公式

```
可激活等级上限 = 玩家等级 + 3
即：entry.monster_level <= player_level + 3
```

适用于普通怪、精英怪、Boss。不使用slot解锁表。

### 3.2 示例

| 玩家等级 | 可激活等级上限 | 效果 |
|---|---|---|
| Lv1 | Lv4 | 激活Lv1-Lv4 entry |
| Lv6 | Lv9 | 激活Lv1-Lv9 entry |
| Lv10 | Lv13 | 激活Lv1-Lv13 entry |
| Lv16 | Lv19 | 普通怪Lv1-Lv19基本全部开放 |
| Lv17 | Lv20 | Boss Lv20可开放 |

### 3.3 禁止错误实现

禁止 slot_unlock_level、Slot1 Lv1解锁、Slot2 Lv3解锁 等写死规则。必须用通用公式。

| 精英怪 | 3 | elite_wolf | Lv11/Lv14/Lv17 |
| Boss | 1 | boss_forest_boar | Lv20 |

总目标：90 + 3 + 1 = 94。

### 4.2 等级生成方式

每个entry的monster_level创建后固定，不随玩家等级变化，不在运行时因刷新改变身份。复活是同一个entry恢复，不是新怪替换。禁止每次复活重新随机成另一个等级。

## 5. WorldEntry 数据模型

建议字段：entry_id / monster_id / monster_role / normal_slot / monster_level / is_elite / is_boss / max_hp / current_hp / attack / defense / gold_reward / state / activation_state / unlocked / world_x / spawn_x / respawn_timer / active_instance

entry_id一旦生成不可重建成新身份。禁止duplicate()复制entry。禁止死亡后生成随机新精英替代。禁止复活后变成另一个等级。

状态规则：

| 状态/字段 | 含义 |
|---|---|
| locked/unlocked=false | 等级未开放，存在但不实例化 |
| active | 已开放并实例化，可巡逻/显示/接战 |
| sleeping | 已开放但临时休眠/离场，可恢复 |
| dead | 已死亡，等待respawn_timer |

locked不是dead/sleeping。locked不参与respawn/target scan/删除。玩家升级后重新评估locked entry。

non_engage不是怪物状态，不进入状态机，不写入存档。是玩家AI选择目标时的即时判断结果。sleeping才是生态系统里的持久休眠状态。

## 6. 生态初始化与激活

初始化：build_all_entries_shuffled(MAP_WIDTH) → 创建94个WorldEntry → shuffle全部entry → 均匀分布world_x/spawn_x到100~3500 → 根据player_level调用_update_ecology_activation(player_level) → 只实例化已开放entry。

分布规则：94个entry均匀分布在100~3500。先创建全部entry→先shuffle→再分段均匀分配world_x。不按物种顺序铺点。Boss/精英有固定位置。locked entry也有固定位置，只是暂不实例化。

激活更新：每次玩家等级变化后调用。如果locked entry满足公式：设置unlocked=true，允许实例化，不改变entry_id/monster_id/monster_level/world_x/spawn_x。不要在玩家视野内无动画弹出。

## 7. 普通怪生命周期

| 阶段 | 规则 |
|---|---|
| 巡逻 | 围绕patrol_origin/spawn_x活动 |
| 感知 | 只有玩家进入aggro_range才能主动追击 |
| 追击 | 进入chase前必须拿到普通attacker slot |
| 攻击 | 进入attack前必须仍在slot内 |
| 放弃 | 超出leash_range必须释放slot并回巡逻 |
| 没拿到slot | 不允许继续贴身挤玩家，应巡逻/等待 |

接战上限：MAX_ATTACKERS = 3（最多3只普通怪主动追击/攻击，不是最多显示3只）。

死亡与复活：respawn_timer = 3.0~8.0秒。locked entry不参与respawn。复活不允许无动画地在玩家眼前凭空弹出（延迟复活或播放出现动画）。

---

## 8. 精英怪生命周期

精英总entry = 3，真实存在 = 3，显示数量由等级门控与实例化决定，接战数量最多1。

禁止：MAX_ELITE_ON_SCREEN、MAX_ACTIVE_ELITE_NEAR_PLAYER。精英未开放时不显示。已开放且在视野内就应该显示。

MAX_ELITE_ATTACKERS = 1（最多1只精英主动追击/攻击，不是最多1只精英存在/显示）。

精英死亡：respawn_timer = 120.0秒。每只精英独立冷却。不走普通3~8秒复活。复活是同一个entry恢复，不是新精英替换。locked精英不参与respawn。

精英等级与开放：elite_wolf Lv11/Lv14/Lv17。激活公式：elite_level <= player_level + 3。

---

## 9. Boss 生命周期

Boss是独立层：数量1，不占普通生态池/精英池，不随玩家等级成长，永不显示MISS。

开放规则：Boss也使用 monster_level <= player_level + 3。彩虹草原Boss Lv20，Lv1-Lv16 locked，Lv17+ unlocked。

目标优先级：先过滤unlocked+alive+can_engage，再按Boss>Elite>Normal排序。Boss存在就无条件攻击Boss是错误的。

彩虹草原Boss建议数值：HP 3000 / ATK 130 / DEF 80 / 攻击间隔 2.2秒 / Lv20。

## 10. 等级感知规则

```
等级差 = 怪物等级 - 玩家等级
```

| 等级差 | 行为 |
|---|---|
| <= -10 | 怪物害怕逃跑 |
| -9 ~ -6 | 玩家无视，不主动攻击 |
| -5 ~ +3 | 正常挂机目标，默认自动攻击 |
| +4 ~ +6 | 危险目标，默认绕开 |
| +7 ~ +9 | 高危目标，默认绕开 |
| >= +10 | 太危险，强制绕开/后退 |

non_engage是即时AI判断，不影响怪物状态。sleeping才是持久休眠状态。

## 12. 玩家出生与探索边界

每次打开游戏固定出生x=100。探索边界：PLAYER_MIN_X=100，PLAYER_MAX_X=3500。玩家自动探索/巡逻/追击/复活都clamp到100~3500。没有合适目标时继续探索，到右边界后折返。

---

## 13. 玩家死亡/复活生态规则

玩家死亡：不清空_enemy_pool/_world_entries/_elite_entries/_boss_entries。不queue_free全场enemy。清空player.target/enemy.target。释放普通/精英attacker slots。播放defeat/death动画。保持不透明，不应变黑影/透明残留。

玩家复活：原地复活（不回x=100）。不teleport到地图中心。摄像机不甩动。2秒invincible + 2秒respawn_grace。grace期间怪物不应立刻重新锁定/攻击玩家。

## 14. 攻防与受击反馈

基础公式：防御比 = 被攻击者防御 ÷ 攻击者攻击。

普通单位受击表：

| 防御比 | 结果 | 表现 | 伤害倍率 |
|---|---|---|---|
| < 0.30 | 重创 | 大后仰、僵直、攻击被打断 | 100% |
| 0.30~0.55 | 压制 | 小后仰、短暂停顿 | 80% |
| 0.55~0.90 | 命中 | 只掉血，不明显后仰 | 60% |
| 0.90~1.40 | 扛住 | 原地承受，少量掉血 | 35% |
| 1.40~2.40 | 硬扛 | 很少掉血 | 15% |
| > 2.40 | 化解/弹开 | 普通小怪攻击玩家时可MISS | 0或1点 |

Boss攻击玩家受击表：

| 防御比 | 结果 | 伤害倍率 |
|---|---|---|
| < 0.35 | Boss重创 | 100% |
| 0.35~0.65 | Boss压制 | 80% |
| 0.65~1.00 | Boss命中 | 60% |
| 1.00~1.60 | 扛住Boss | 35% |
| 1.60~2.80 | 硬扛Boss | 15% |
| > 2.80 | 化解Boss | 1点或5% |

Boss永远不显示MISS。

## 19. 药水规则

药水为掉落产物（不可购买），用于避免频繁死亡。

| 来源 | 规则 |
|---|---|
| 普通怪掉落 | 随装备一并掉落，掉率与装备同级 |
| 精英怪掉落 | 掉率高于普通怪 |
| Boss掉落 | 100%掉落1瓶 |
| 商店购买 | 不支持 |

自动用药：HP低于40%自动使用。冷却8秒。单瓶恢复35%最大生命。背包上限10瓶。死亡不丢装备/金币。

## 20. 验收标准

1. Godot启动无报错
2. 94个WorldEntry真实存在（普通90+精英3+Boss1）
3. 普通怪由5个normal slot组成，等级覆盖1-6/4-10/7-13/10-16/13-19
4. 激活公式为 monster_level <= player_level + 3
5. Lv1只激活Lv1-Lv4 entry，Lv17后Boss Lv20才可开放
6. 没有同屏上限控制存在，没有屏幕刷怪残留
7. 玩家默认只攻击等级差-5到+3的怪
8. 普通怪最多3只接战，精英怪最多1只接战
9. 玩家死亡不重置生态，复活安全靠invincible/grace
10. 精英死亡后per-entry 120秒冷却
11. Boss永远不显示MISS
12. 装备掉落从固定词条库生成
