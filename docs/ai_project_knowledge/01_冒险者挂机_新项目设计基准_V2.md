## 3. 游戏定位

冒险者挂机 是一款 **2D 横版桌面挂机 RPG**。

玩家在桌面上看到一个透明挂机窗口。角色在小型横版战斗条中自动战斗、自动成长、自动拾取资源。玩家主要通过装备、词条、图鉴、宠物、时装、地图推进和长期挂机收益获得成长。

本项目不是：

- 传统手操动作游戏。
- 复杂开放世界。
- 多人联网游戏。
- 商城驱动游戏。
- 大型剧情 RPG。

本项目是：

- 桌面小窗口挂机游戏。
- 角色长期成长游戏。
- 刷装备与词条游戏。
- 收集图鉴与宠物游戏。
- 轻量但可长期玩的单机游戏。

---

## 4. 核心体验

玩家打开游戏后，桌面上出现一个透明小窗口。默认状态下主要看到战斗条。角色在战斗条中自动打怪，获得经验、金币、装备、材料、图鉴进度和宠物养成资源。

玩家的主要操作不是频繁手动战斗，而是：

- 查看掉落装备。
- 更换装备。
- 洗练词条。
- 查看幻化外观变化。
- 查看图鉴进度。
- 培养宠物。
- 切换地图。
- 管理仓库。
- 调整挂机位置。

核心循环：

```text
自动寻找怪物
→ 自动战斗
→ 怪物死亡
→ 掉落金币 / 经验 / 装备 / 材料
→ 图鉴记录推进
→ 玩家属性成长
→ 装备和宠物变强
→ 解锁更高地图或更强怪物
→ 持续挂机刷长期目标
```

## 6. 平台与窗口总规则（V2 已纠偏）

第一阶段只做 Windows 桌面版。

本项目不再采用旧的“战斗条为锚点的透明动态布局系统”。

旧方案废弃：

```text
战斗条是主锚点
三栏围绕战斗条展开
拖动战斗条带动三栏
三栏和战斗条共同参与穿透区域计算
```

新方案固定为：

```text
1440×720 透明逻辑画布
WindowShell 统一管理窗口尺寸、拖动、透明、鼠标穿透
常驻 BattleWidget 挂机条（不从 PageHost 切换，始终保持 visible）
CenterPanel 核心管理界面（从 BattleBar 打开，左半角色属性 + 右半背包）
LeftDockHost / RightDockHost 扩展容器（内容由 CenterPanel 按钮决定）
DockLayoutController 统一管理布局与翻转/让位
```

正确窗口结构：

```text
MainWindow 1440×720 transparent logic canvas
  WindowShell
    TransparentCanvas
    WindowDragLayer
    InputRegionManager
    DockLayer
      LeftDockHost          ← 左侧扩展容器，内容由 CenterPanel 按钮决定
        [动态内容: 仓库/宠物/图鉴/地图]
      CenterDockHost        ← 核心管理界面
        CenterPanel
          ├── 左半：角色/属性/装备穿戴
          └── 右半：背包
      RightDockHost         ← 右侧扩展容器，内容由 CenterPanel 按钮决定
        [动态内容: 设置/洗练/详情]
    BattleWidget
      TopInfoBar
      HpBar
      ExpBar
      GoldLabel
      BagButton
```

核心原则：

- BattleWidget / BattleBar 是常驻挂机条，始终保持显示。
- CenterPanel 从 BattleBar.BagButton 打开，宽度与 BattleBar 一致。
- CenterPanel 是核心管理界面：左半（角色/属性/装备穿戴）+ 右半（背包）。
- LeftPanel / RightPanel 是扩展容器，内容由 CenterPanel 内按钮决定。
- 布局与翻转/让位由 DockLayoutController 统一管理。
- 拖动由 BattleWidget._gui_input 自行处理，通过 battle_widget_moved 信号通知 DockLayoutController 重新 relayout。
- 鼠标穿透由 InputRegionManager 统一负责。
- 任何面板不能直接控制全局窗口穿透、窗口尺寸、窗口拖动。

错误方案：

```text
固定小战斗条作为唯一主锚点
从战斗条展开三栏
拖动战斗条时三栏跟随
切页面时改变窗口尺寸
页面脚本自己计算鼠标穿透区域
```

这些方案禁止继续使用。

## 9. 透明与鼠标穿透规则（新架构）

### 9.1 透明区域

除实际可见 UI、页面、按钮、战斗条、展开弹窗外，其余区域可以透明。

主窗口采用 1440×720 固定逻辑尺寸。透明区域由 WindowShell / InputRegionManager 统一计算。

禁止：

- 页面脚本自己调用全局 `window_set_mouse_passthrough`。
- BattleStrip 自己控制全局穿透。
- 三栏和战斗条合并计算穿透区域。
- 透明区域永久吃鼠标。

### 9.2 可实现穿透方式

项目采用 Godot 4.7 的 DisplayServer 方式：

```gdscript
DisplayServer.window_set_flag(DisplayServer.WINDOW_FLAG_TRANSPARENT, true)
DisplayServer.window_set_mouse_passthrough(region)
```

`region` 是窗口内允许接收鼠标事件的多边形区域。区域外鼠标会穿透到后面的桌面或窗口。

正式实现必须由：

```text
res://scripts/window/input_region_manager.gd
```

统一收集可交互区域，并统一调用 DisplayServer。

### 9.3 输入区域来源

InputRegionManager 只收集：

- BattleWidget 可交互矩形。
- CenterPanel 当前可见交互矩形。
- LeftPanel 当前可见面板交互矩形。
- RightPanel 当前可见面板交互矩形。
- WindowDragLayer 可拖动区域。
- 当前打开弹窗的可交互控件。

不收集：

- 隐藏页面。
- 透明装饰。
- 不可点击背景。
- 纯视觉特效。

## 10. 拖动规则（新架构）

窗口拖动由 WindowShell 统一处理。

允许拖动区域：

- 标题栏空白。
- WindowDragLayer。
- 非交互面板背景。

禁止拖动区域：

- 标签按钮。
- 背包格子。
- 装备槽。
- 地图卡片。
- 滚动条。
- 设置开关。
- 技能图标。
- 物品图标。

拖动不能由 BattleStrip 独占。

这是新项目最高优先级规则。

凡是玩家能看到的东西，都必须优先在 Godot 中做真实可视化资源。

### 12.1 必须可视化的内容

以下内容必须用 `.tscn`、`Sprite2D`、`AnimatedSprite2D`、`Control`、`TextureRect`、`AnimationPlayer` 等 Godot 可视化方式制作：

- UI 面板。
- 战斗条。
- 按钮。
- 物品格。
- 装备格。
- 图鉴界面。
- 宠物界面。
- 幻化角色。

## 15. 战斗系统

### 15.1 战斗方式

战斗为自动战斗。

玩家不需要手动攻击。

### 15.2 基础战斗流程

```text
玩家扫描目标
→ 判断是否接战
→ 移动到攻击距离
→ 自动攻击
→ 怪物受伤
→ 怪物死亡
→ 掉落奖励
→ 选择下一个目标
```

### 15.3 non_engage 定义

`non_engage` 中文含义：不接战 / 不攻击 / 绕开。

规则：

```text
non_engage 不是怪物状态。
non_engage 不进入状态机。
non_engage 不写入存档。
non_engage 只是玩家 AI 选择目标时的即时判断结果。
```

当怪物等级过低或过高时，玩家 AI 可以跳过它。下次扫描目标时重新判断。

### 15.4 sleeping 定义

`sleeping` 中文含义：休眠。

`sleeping` 是生态系统里的持久状态。

规则：

```text
sleeping 是怪物或生态 entry 的状态。
sleeping 可以持续一段时间。
sleeping 可以影响生态刷新。
sleeping 可以被系统恢复。
```

## 16. 生态与掉落系统

完整规则以新项目系统文档为准：

```text
docs/ai_project_knowledge/09_冒险者挂机_彩虹草原生态战斗掉落基准_V2.md
```

禁止引用旧项目生态掉落文档。

### 16.1 Slot 定义

`Slot` 是当前地图内部怪物分组。

即使第二张地图 Slot 1 是 Lv21 怪，它在第二张地图内部仍然是新手怪。

因此：

```text
每张地图内部 Slot 1 掉装反馈更高。
每张地图内部 Slot 5 掉装总率更低，但品质更高。
```

这是刻意设计，不是写反。

### 16.2 掉落两层判定

掉落系统分两层：

```text
第一层：是否掉装备。
第二层：如果掉了装备，再决定装备品质。
```

### 16.3 普通怪掉率方向

普通怪 Slot 1-5 全部允许红装掉落。

普通怪总装备掉率采用当前地图内反向曲线：

```text
Slot 1 掉装最高。
Slot 5 掉装最低。
```

后期虽然总掉装更低，但品质概率更高。

### 16.4 精英怪与 Boss

精英怪和 Boss 必须有金装、红装概率。

不能只有 Boss 才有红装惊喜。

Boss 应该是阶段性奖励点，掉落体验明显强于普通怪与精英怪。

## 17. 装备系统

完整规则以新项目系统文档为准：

```text
docs/ai_project_knowledge/08_冒险者挂机_装备词条库_Lv1-Lv20_T1-T4_V2.md
```

禁止引用旧项目装备词条文档。

### 17.1 装备系统总原则

- 装备有品质。
- 装备有部位。
- 装备有词条。
- 装备可掉落。
- 装备可洗练。
- 装备可穿戴。
- 装备影响属性，不直接绑定角色外观。外观由独立幻化系统控制。
- 装备是长期成长核心。

### 17.2 百分比词条规则

装备词条不设百分比词条数量上限。

允许极品装备出现多个百分比词条。

但同一个 `mod_group` 不允许重复出现。

固定值词条和百分比词条可以同时出现。

例如：

```text
flat_attack
percent_attack
```

可以同时出现在一件装备上。

### 17.3 装备与幻化关系

装备系统和幻化系统各自独立。装备负责属性、品质、词条、掉落、穿戴、洗练。幻化负责角色整套外观显示。

装备不绑定外观资源。幻化不绑定战斗属性。换装备不影响角色外观。换幻化不影响战斗数值。

## 18. 人物幻化系统

人物幻化系统是新项目核心系统之一。

### 18.1 目的

人物幻化系统用于控制角色整体外观。装备影响属性，不直接影响角色外观。角色外观由独立的整套幻化皮肤控制。幻化影响显示，不影响核心战斗数值。

### 18.2 幻化资源规则

每个幻化是一整套角色动画资源包。每套幻化至少包含以下动画：

```text
idle / run / attack / hurt / die
```

幻化资源按 skin_id 管理：

```text
skin_novice_001
skin_swordsman_001
skin_grass_traveler_001
```

Godot 中通过替换整套 AnimatedSprite2D / SpriteFrames 资源实现外观切换。不使用纸娃娃分层叠加。

### 18.3 幻化可视化要求

幻化资源必须在 Godot 中可视化。每个 skin_id 对应的整套 SpriteFrames / AnimatedSprite2D 资源必须在编辑器中可见、可预览。

禁止用纯代码拼角色外观。

## 20. 图鉴系统

图鉴系统是新项目核心成长系统之一。

图鉴分为：

- 怪物图鉴。
- 掉落图鉴。
- 消耗品图鉴。
- 宠物图鉴。
- 装备图鉴。
- 时装图鉴。

第一版优先做：

- 怪物图鉴。
- 掉落图鉴。
- 消耗品图鉴。

### 20.1 怪物图鉴

玩家击杀怪物后，怪物图鉴记录进度。

记录内容：

- 怪物名称。
- 怪物等级。
- 怪物类型。
- 已击杀数量。
- 首次击杀时间。
- 掉落物列表。
- 是否已解锁。

击杀数量达到阶段后可以提供属性加成。

示例：

```text
击杀 10 只：解锁怪物资料。
击杀 100 只：生命 +1。
击杀 500 只：攻击 +1。
击杀 1000 只：对该怪物伤害 +1%。
```

### 20.2 掉落图鉴

怪物掉落过的物品会进入掉落图鉴。

记录内容：

- 物品名称。
- 来源怪物。
- 来源地图。
- 首次获得时间。
- 获得次数。
- 最高品质。
- 是否已收集。

### 20.3 消耗品图鉴

消耗品使用后，可以激活图鉴记录。

例如：

```text
使用蘑菇孢子。
激活蘑菇孢子图鉴。
获得永久生命 +1。
```

消耗品图鉴规则：

- 第一次使用激活图鉴。
- 重复使用增加图鉴经验。
- 图鉴升级获得永久属性。
- 图鉴属性为长期成长来源。
