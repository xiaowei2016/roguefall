## 第一部分：人物幻化与时装系统

### 1.1 核心原则

装备影响属性，不直接影响角色外观。角色外观由独立的整套幻化皮肤控制。幻化影响显示，不影响核心战斗数值。

### 1.2 幻化资源规则

每个幻化是一整套角色动画资源包。每套幻化至少包含以下动画：

```text
idle / run / attack / hurt / die
```

幻化资源按 skin_id 管理：

```text
skin_novice_001
skin_swordsman_001
skin_grass_traveler_001
```

Godot 中通过替换整套 AnimatedSprite2D / SpriteFrames 资源实现外观切换。不使用纸娃娃分层叠加。

### 1.3 幻化切换规则

玩家可在幻化界面中选择当前使用的整套皮肤。切换幻化时，角色整套动画资源全部替换，不涉及单层或单件混合。

幻化不绑定任何装备属性、词条、品质或掉落来源。幻化只决定角色在战斗条中显示的外观。

### 1.4 时装系统

时装系统是幻化系统的一部分。时装有独立的收藏属性加成，但不成为主要战斗数值来源。

### 1.5 可视化规则

幻化资源必须在 Godot 中可视化。每个 skin_id 对应的整套 SpriteFrames / AnimatedSprite2D 资源必须在编辑器中可见、可预览。禁止用纯代码拼角色外观。

## 第二部分：图鉴系统

### 2.1 图鉴类型

- 怪物图鉴
- 掉落图鉴
- 消耗品图鉴
- 装备图鉴
- 宠物图鉴
- 时装图鉴

第一版优先做：怪物图鉴、掉落图鉴、消耗品图鉴。

### 2.2 怪物图鉴

玩家击杀怪物后，怪物图鉴记录进度。

记录内容：怪物名称、怪物等级、怪物类型、已击杀数量、首次击杀时间、掉落物列表、是否已解锁。

击杀数量达到阶段后提供属性加成：

```text
击杀 10 只：解锁怪物资料。
击杀 100 只：生命 +1。
击杀 500 只：攻击 +1。
击杀 1000 只：对该怪物伤害 +1%。
```

### 2.3 掉落图鉴

怪物掉落过的物品会进入掉落图鉴。

记录内容：物品名称、来源怪物、来源地图、首次获得时间、获得次数、最高品质、是否已收集。

### 2.4 消耗品图鉴

消耗品使用后可以激活图鉴记录。

规则：
- 第一次使用激活图鉴
- 重复使用增加图鉴经验
- 图鉴升级获得永久属性
- 图鉴属性为长期成长来源

## 第三部分：宠物系统

### 3.1 宠物定位

宠物提供：属性加成、收集目标、跟随表现、长期养成。

### 3.2 多宠物携带

计划支持携带多只宠物。初版设计：

```text
最多携带 3 只宠物。
其中 1 只是主宠物。
其余为辅助宠物。
```

主宠物：显示在战斗页中、提供完整属性加成、可有跟随表现。
辅助宠物：可以不显示或简化显示、提供部分属性加成。

### 3.3 宠物属性

宠物可以提供：生命、攻击、防御、攻速、暴击率、金币加成、经验加成、掉落率加成、Boss 伤害、精英伤害。

### 3.4 宠物可视化

宠物必须有 Godot 可视化资源。宠物图标、宠物跟随、宠物界面都不能只用代码生成。
