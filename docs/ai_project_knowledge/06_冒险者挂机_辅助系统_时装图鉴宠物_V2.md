## 第一部分：纸娃娃与时装系统

### 1.1 目的

纸娃娃系统用于让装备变化体现在角色外观上。玩家换装备后，不只是属性变化，也能看到角色外观变化。

### 1.2 纸娃娃层级

角色外观由多个层组成：

```text
基础身体层 → 发型层 → 脸部层 → 武器层 → 头部装备层 →
身体装备层 → 鞋子层 → 背部/披风层 → 饰品层 → 时装覆盖层 → 特效层
```

推荐节点结构：

```text
PlayerPaperDoll.tscn
  BodyLayer
  HairLayer
  FaceLayer
  WeaponLayer
  HelmetLayer
  ArmorLayer
  ShoesLayer
  BackLayer
  AccessoryLayer
  CostumeLayer
  EffectLayer
```

### 1.3 装备外观绑定

每件可视化装备都可以绑定：图标、纸娃娃外观资源、掉落来源、品质、词条、装备等级。

换武器时：属性变化 + 武器图层变化。
换衣服时：属性变化 + 身体装备图层变化。

### 1.4 时装覆盖规则

时装系统独立于装备属性。

```text
实际装备仍然是 armor_010。
显示外观使用 costume_body_003。
```

时装优先级高于装备外观。

### 1.5 可视化规则

纸娃娃层级必须在 Godot 中可视化。禁止用纯代码拼角色外观。

## 第二部分：图鉴系统

### 2.1 图鉴类型

- 怪物图鉴
- 掉落图鉴
- 消耗品图鉴
- 装备图鉴
- 宠物图鉴
- 时装图鉴

第一版优先做：怪物图鉴、掉落图鉴、消耗品图鉴。

### 2.2 怪物图鉴

玩家击杀怪物后，怪物图鉴记录进度。

记录内容：怪物名称、怪物等级、怪物类型、已击杀数量、首次击杀时间、掉落物列表、是否已解锁。

击杀数量达到阶段后提供属性加成：

```text
击杀 10 只：解锁怪物资料。
击杀 100 只：生命 +1。
击杀 500 只：攻击 +1。
击杀 1000 只：对该怪物伤害 +1%。
```

### 2.3 掉落图鉴

怪物掉落过的物品会进入掉落图鉴。

记录内容：物品名称、来源怪物、来源地图、首次获得时间、获得次数、最高品质、是否已收集。

### 2.4 消耗品图鉴

消耗品使用后可以激活图鉴记录。

规则：
- 第一次使用激活图鉴
- 重复使用增加图鉴经验
- 图鉴升级获得永久属性
- 图鉴属性为长期成长来源

## 第三部分：宠物系统

### 3.1 宠物定位

宠物提供：属性加成、收集目标、跟随表现、长期养成。

### 3.2 多宠物携带

计划支持携带多只宠物。初版设计：

```text
最多携带 3 只宠物。
其中 1 只是主宠物。
其余为辅助宠物。
```

主宠物：显示在战斗页中、提供完整属性加成、可有跟随表现。
辅助宠物：可以不显示或简化显示、提供部分属性加成。

### 3.3 宠物属性

宠物可以提供：生命、攻击、防御、攻速、暴击率、金币加成、经验加成、掉落率加成、Boss 伤害、精英伤害。

### 3.4 宠物可视化

宠物必须有 Godot 可视化资源。宠物图标、宠物跟随、宠物界面都不能只用代码生成。
